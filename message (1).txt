# Tournament Bot - Complete Development Specification

## Project Goal

Build a professional Discord Tournament Management Bot using **discord.js (latest)** with a modern UI based on Slash Commands, Buttons, Select Menus, Modals and Dynamic Embeds. The bot should automate the complete tournament registration workflow while keeping payment verification manual.

The system must be modular, scalable, editable and support multiple tournaments running simultaneously.

---

# 1. Tournament Creation

## Slash Command

```
/match create
```

Only administrators or users with configured permissions can use this command.

The command should launch a multi-step setup wizard instead of asking everything at once.

# 31. Overall Workflow

1. Admin runs `/match create`.
2. Bot opens a setup wizard and creates a private preview embed.
3. Admin configures tournament details, visuals, registration fee, currency, slots, timer, reminders, and optional prize pool.
4. Admin publishes the finalized embed to the tournament announcement channel.
5. Users click **Register** and complete the registration modal.
6. A private ticket channel is created automatically, where the bot posts the QR code and payment instructions.
7. The user uploads a payment screenshot.
8. Admin manually approves or rejects the registration.
9. On approval, the slot count, remaining slots, countdown, and total collected amount update live on the public embed.
10. When the timer expires or all slots are filled, registration closes automatically.
11. If enabled, the bot prompts the admin to configure the prize distribution.
12. Before the tournament starts, the bot reminds the admin (1 hour before) and sends configurable DM reminders to all approved participants (e.g., 20 or 15 minutes before).
13. Admin releases the Room ID and Password, which are automatically DM'd to all approved players.
14. All actions are logged, and the tournament remains editable through dedicated management commands until it is completed or cancelled.
